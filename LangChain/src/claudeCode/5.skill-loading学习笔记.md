# LangChain Skill Loading 学习笔记

## 概述

本文通过分析 `5.skill-loading.ts` 文件，学习渐进式技能加载（Progressive Skill Loading）模式。**核心洞察：不要用未使用的技能膨胀上下文。**

## 核心架构

### 双层加载模式

```
System prompt                      Tool call
+--------------------------+      +------------------+
| Available skills:        |      | skill("xyz")     |
|   - xyz: description     | ---> | <skill>          |
|   - abc: description     |      |   full content   |
+--------------------------+      | </skill>         |
                                  +------------------+
```

### 与 s04 的区别

| 方面 | s04 (subagent) | s05 (skill-loading) |
|------|----------------|---------------------|
| 工具数量 | 6个 | **5个** (-task, +skill) |
| 知识注入 | 无 | **双层渐进** |
| System Prompt | 固定 | **动态生成** |
| 上下文策略 | 隔离 | **按需加载** |
| 知识来源 | 模型内置 | **外部文件** |


**设计思想：**
- `meta` 用于快速浏览和选择
- `body` 用于详细指导和执行
- 分离元数据与内容，便于按需加载

### 2. SkillLoader 类

```typescript
class SkillLoader {
  private skillsDir: string;
  private skills: Map<string, Skill> = new Map();

  constructor(skillsDir: string) {
    this.skillsDir = skillsDir;
    this.loadAll();  // 初始化时加载所有技能元数据
  }

  private loadAll(): void {
    if (!fs.existsSync(this.skillsDir)) {
      return;
    }

    const skillFiles = this.findSkillFiles(this.skillsDir);
    for (const filePath of skillFiles.sort()) {
      const text = fs.readFileSync(filePath, "utf-8");
      const { meta, body } = this.parseFrontmatter(text);
      const name = meta.name || path.dirname(filePath).split("/").pop() || filePath;
      this.skills.set(name, { meta, body, path: filePath });
    }
  }

  private findSkillFiles(dir: string): string[] {
    const results: string[] = [];
    const entries = fs.readdirSync(dir, { withFileTypes: true });

    for (const entry of entries) {
      const fullPath = path.join(dir, entry.name);
      if (entry.isDirectory()) {
        results.push(...this.findSkillFiles(fullPath));  // 递归搜索
      } else if (entry.name === "SKILL.md") {  // 只识别 SKILL.md
        results.push(fullPath);
      }
    }
    return results;
  }
}
```

**加载流程：**
```
skills/
├── code-review/
│   └── SKILL.md ← 被识别
├── testing/
│   └── SKILL.md ← 被识别
└── deployment/
    └── SKILL.md ← 被识别

↓ loadAll()

skills Map:
├── "code-review" → {meta, body, path}
├── "testing" → {meta, body, path}
└── "deployment" → {meta, body, path}
```

### 3. YAML Frontmatter 解析

```typescript
private parseFrontmatter(text: string): { meta: SkillMeta; body: string } {
  // 匹配 YAML frontmatter 模式
  const match = text.match(/^---\n([\s\S]*?)\n---\n([\s\S]*)$/);
  
  if (!match) {
    return { meta: { name: "", description: "No description" }, body: text };
  }

  const meta: SkillMeta = { name: "", description: "No description" };
  const yamlContent = match[1]!;
  const lines = yamlContent.split("\n");
  const parsed: Record<string, string> = {};

  // 简单的 YAML 解析（仅支持 key: value）
  for (const line of lines) {
    const colonIndex = line.indexOf(":");
    if (colonIndex > 0) {
      const key = line.slice(0, colonIndex).trim();
      const value = line.slice(colonIndex + 1).trim().replace(/^["']|["']$/g, "");
      parsed[key] = value;
    }
  }

  meta.name = parsed.name || "";
  meta.description = parsed.description || "No description";
  if (parsed.version) meta.version = parsed.version;
  if (parsed.tags) meta.tags = parsed.tags;

  return { meta, body: match[2]!.trim() };
}
```

### 4. 双层获取方法

```typescript
// Layer 1: 简短描述（用于系统提示）
getDescriptions(): string {
  if (this.skills.size === 0) {
    return "(no skills available)";
  }

  const lines: string[] = [];
  for (const [name, skill] of this.skills) {
    const desc = skill.meta.description || "No description";
    let line = `  - ${name}: ${desc}`;
    if (skill.meta.tags) {
      line += ` [${skill.meta.tags}]`;
    }
    lines.push(line);
  }
  return lines.join("\n");
}

// Layer 2: 完整内容（按需加载）
getContent(name: string): string {
  const skill = this.skills.get(name);
  if (!skill) {
    return `Error: Unknown skill '${name}'. Available: ${Array.from(this.skills.keys()).join(", ")}`;
  }
  return `<skill name="${name}">\n${skill.body}\n</skill>`;
}
```

## 核心设计模式

### 1. 渐进式信息加载

```
启动时                  运行时
┌─────────────┐        ┌─────────────┐
│ Load Meta   │        │ Load Body   │
│ (元数据)    │        │ (完整内容)  │
└──────┬──────┘        └──────┬──────┘
       │                      │
       ▼                      ▼
┌─────────────┐        ┌─────────────┐
│ Inject to   │        │ Return via  │
│ System Prompt│        │ Tool Result │
└─────────────┘        └─────────────┘
       │                      │
       ▼                      ▼
   始终可见              按需可见
   (Layer 1)            (Layer 2)
```

**Token 消耗对比：**
```
场景：10个技能，每个500 tokens

传统方式（全部注入）:
  System Prompt: 10 × 500 = 5000 tokens

渐进加载（双层）:
  System Prompt: 10 × 20 = 200 tokens (仅描述)
  Skill 工具调用: 500 tokens (按需加载一个)
  
节省: 5000 - 700 = 4300 tokens (86%)
```

## 关键学习点

### 1. Token 经济学

| 策略 | Token 消耗 | 适用场景 |
|------|-----------|----------|
| 全量注入 | 高 | 技能数量少、频繁使用 |
| **渐进加载** | 低 | 技能数量多、使用不频繁 |
| 混合模式 | 中 | 核心技能注入 + 其他按需 |

**决策依据：**
```
if (技能数量 < 5 && 平均每轮使用 > 2个) {
  策略 = "全量注入";
} else if (技能数量 > 10) {
  策略 = "渐进加载";
} else {
  策略 = "混合模式";
}
```

### 2. 知识组织方式

| 方式 | 优点 | 缺点 |
|------|------|------|
| 硬编码 | 简单直接 | 难以维护 |
| 配置文件 | 灵活 | 需要解析 |
| **文件约定** | 自发现 | 命名限制 |
| 数据库 | 动态、可扩展 | 复杂、依赖重 |

**文件约定的优势：**
- 新增技能：创建目录 + SKILL.md 文件
- 删除技能：删除目录
- 版本控制：Git 友好
- 无需重启：自动重载（如果实现）

### 2. 技能依赖管理

```typescript
interface SkillMeta {
  name: string;
  requires?: string[];  // 依赖的其他技能
}

getContent(name: string): string {
  const skill = this.skills.get(name);
  if (!skill) return `Error: Unknown skill '${name}'`;
  
  // 递归加载依赖
  const contents: string[] = [];
  const loaded = new Set<string>();
  
  const loadWithDeps = (skillName: string) => {
    if (loaded.has(skillName)) return;
    loaded.add(skillName);
    
    const s = this.skills.get(skillName);
    if (!s) return;
    
    // 先加载依赖
    if (s.meta.requires) {
      for (const dep of s.meta.requires) {
        loadWithDeps(dep);
      }
    }
    
    contents.push(`<skill name="${skillName}">\n${s.body}\n</skill>`);
  };
  
  loadWithDeps(name);
  return contents.join("\n\n");
}
```

### 3. 技能版本管理

```typescript
interface SkillMeta {
  name: string;
  version?: string;
}

class SkillLoader {
  private versions: Map<string, Skill[]> = new Map();
  
  loadAll(): void {
    // ... 加载所有版本
    for (const skill of skills) {
      if (!this.versions.has(skill.meta.name)) {
        this.versions.set(skill.meta.name, []);
      }
      this.versions.get(skill.meta.name)!.push(skill);
    }
  }
  
  getContent(name: string, version?: string): string {
    const skills = this.versions.get(name);
    if (!skills) return `Error: Unknown skill '${name}'`;
    
    // 选择版本
    const skill = version 
      ? skills.find(s => s.meta.version === version)
      : skills[skills.length - 1];  // 默认最新
    
    if (!skill) return `Error: Version '${version}' not found`;
    
    return `<skill name="${name}" version="${skill.meta.version}">\n${skill.body}\n</skill>`;
  }
}
```

```typescript
class SkillAnalytics {
  private usageCount: Map<string, number> = new Map();
  
  trackLoad(skillName: string): void {
    this.usageCount.set(skillName, 
      (this.usageCount.get(skillName) || 0) + 1
    );
  }
  
  getTopSkills(n: number): string[] {
    return Array.from(this.usageCount.entries())
      .sort((a, b) => b[1] - a[1])
      .slice(0, n)
      .map(([name]) => name);
  }
  
  getUnusedSkills(): string[] {
    return Array.from(this.usageCount.entries())
      .filter(([_, count]) => count === 0)
      .map(([name]) => name);
  }
}
```

## 总结

### 核心学习点

1. **双层架构**: 描述始终可见，内容按需加载
2. **文件约定**: SKILL.md 文件自动发现，零配置
3. **简化解析**: 简单 YAML，避免复杂依赖

### 设计原则

- **按需原则**: 不用不加载，节省 token
- **约定原则**: 文件命名约定，自动发现
- **简洁原则**: 简化解析，零依赖
- **可扩展**: 支持版本、依赖、分析

### 权衡决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 加载策略 | 双层 | 平衡 token 和可用性 |
| 文件格式 | YAML Frontmatter | 人类可读、工具友好 |
| 解析方式 | 简化实现 | 避免依赖、快速 |
| 技能标识 | 文件名 + meta.name | 灵活性 |
