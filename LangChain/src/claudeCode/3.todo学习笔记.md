## 代码结构分析

### 1. System Prompt

**关键指导：**

- 开始前标记 `in_progress`
- 完成后标记 `completed`
- 优先使用工具而非文字描述

### 2. TodoManager 类

```typescript
class TodoManager {
  items: Array<{ id: string; text: string; status: string }>;

  update(items: Array<{ id: string; text: string; status: string }>): string {
    // 1. 数量限制
    if (items.length > 20) {
      throw new Error("Max 20 todos allowed");
    }

    // 2. 验证并规范化
    const validated: Array<{ id: string; text: string; status: string }> = [];
    let inProgressCount = 0;
    
    for (let i = 0; i < items.length; i++) {
      const item: any = items[i];
      const text = String(item.text ?? "").trim();
      const status = String(item.status ?? "pending").toLowerCase();
      const itemId = String(item.id ?? String(i + 1));

      // 2.1 文本必填
      if (!text) {
        throw new Error(`Item ${itemId}: text required`);
      }

      // 2.2 状态必须有效
      if (!["pending", "in_progress", "completed"].includes(status)) {
        throw new Error(`Item ${itemId}: invalid status '${status}'`);
      }

      // 2.3 统计进行中数量
      if (status === "in_progress") {
        inProgressCount++;
      }

      validated.push({ id: itemId, text, status });
    }

    // 3. 只能有一个任务进行中
    if (inProgressCount > 1) {
      throw new Error("Only one task can be in_progress at a time");
    }

    this.items = validated;
    return this.render();
  }

  render(): string {
    if (this.items.length === 0) {
      return "No todos.";
    }

    const lines: string[] = [];
    for (const item of this.items) {
      const marker = { 
        pending: "[ ]", 
        in_progress: "[>]", 
        completed: "[x]" 
      }[item.status];
      
      lines.push(`${marker} #${item.id}: ${item.text}`);
    }

    const done = this.items.filter((t) => t.status === "completed").length;
    lines.push(`\n(${done}/${this.items.length} completed)`);
    
    return lines.join("\n");
  }
}
```

**设计约束：**
| 约束 | 说明 | 目的 |
|------|------|------|
| 最大20个任务 | `items.length > 20` | 防止任务爆炸 |
| 唯一进行中 | `inProgressCount > 1` | 单焦点执行 |
| 状态必须有效 | `pending/in_progress/completed` | 防止无效状态 |
| 文本必填 | `!text` | 任务必须清晰 |

### 4. Nag 提醒机制

```typescript
const agentLoop = async (messages: BaseMessage[]) => {
  let roundsSinceTodo = 0;  // 计数器

  while (true) {
    // 添加系统消息
    const messagesWithSystem = [
      new SystemMessage({ content: SYSTEM_PROMPT }),
      ...messages,
    ];
    
    const response = await llmWithTools.invoke(messagesWithSystem);
    messages.push(response);
    
    if (response.response_metadata?.finish_reason !== "tool_calls") {
      return;
    }

    const results: (ToolMessage | HumanMessage)[] = [];
    let usedTodo = false;

    // 执行工具调用
    if (response.tool_calls) {
      for (const toolCall of response.tool_calls) {
        const handler = TOOL_HANDLERS[toolCall.name];
        let output: string;

        try {
          output = handler 
            ? handler(toolCall.args) 
            : `Unknown tool: ${toolCall.name}`;
        } catch (error: any) {
          output = `Error: ${error.message}`;
        }

        results.push(new ToolMessage({
          tool_call_id: toolCall.id!,
          content: output,
        }));

        if (toolCall.name === "todo") {
          usedTodo = true;  // 标记使用了 todo
        }
      }
    }

    // 更新计数器
    roundsSinceTodo = usedTodo ? 0 : roundsSinceTodo + 1;

    // Nag 提醒：连续3轮没用 todo 就提醒
    if (roundsSinceTodo >= 3) {
      results.push(new HumanMessage({
        content: "<reminder>Update your todos.</reminder>",
      }));
    }

    messages.push(...results);
  }
};
```

**Nag 机制流程：**
```
轮次  工具调用       roundsSinceTodo  动作
────  ──────────     ─────────────── ───────
1     todo           0 → 0           正常
2     bash           0 → 1           正常
3     read_file      1 → 2           正常
4     write_file     2 → 3           ⚠️ 注入提醒
5     todo           3 → 0           重置
```

## 核心设计模式

### 1. 状态管理模式

```
┌─────────────────┐
│   TodoManager    │  ← 单例，全局状态
│   (内存中)        │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   todo 工具      │  ← 唯一修改入口
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   LLM 调用      │  ← 通过工具更新
└─────────────────┘
```

**设计原则：**
- 单一入口：只能通过 `todo` 工具修改状态
- 状态一致：更新后立即渲染
- 约束保证：验证逻辑在 `update()` 中

### 2. 提醒模式（Nag Pattern）

```typescript
// 检测模型"忘记"的行为
if (roundsSinceTodo >= 3) {
  // 注入人工消息，模拟用户提醒
  results.push(new HumanMessage({
    content: "<reminder>Update your todos.</reminder>",
  }));
}
```

**模式本质：**
- 检测模型行为偏差
- 注入修正信号
- 不改变模型权重，只影响当前上下文

## 关键学习点

### 1. 任务状态机

```
           start
             │
             ▼
         ┌───────┐
      ┌──│pending│
      │  └───────┘
      │       │
      │       │ mark in_progress
      │       ▼
      │  ┌───────────┐
      │  │in_progress│
      │  └───────────┘
      │       │
      │       │ mark completed
      │       ▼
      │  ┌─────────┐
      └──│completed│
         └─────────┘
             │
             ▼
            end
```

**状态转换规则：**
- `pending → in_progress`: 开始执行
- `in_progress → completed`: 执行完成
- `completed → pending`: 回退（特殊场景）
- 不允许跳跃：`pending → completed`（虽然技术上可行，但不推荐）

### 2. System Prompt 的作用

| 元素 | 作用 |
|------|------|
| 身份定义 | "coding agent" - 设定角色 |
| 工具指导 | "Use todo tool" - 指明行为 |
| 规范约束 | "Mark in_progress before starting" - 操作规范 |
| 输出约束 | "Always answer in Chinese" - 语言约束 |

### 3. 错误处理策略

### 2. Nag 阈值选择

| 阈值 | 效果 |
|------|------|
| 1轮 | 过于频繁，干扰模型 |
| 2轮 | 可能过早，打断正常流程 |
| **3轮** | 平衡点，足够容忍又不过度 |
| 5轮+ | 过于宽松，模型可能完全忘记 |

### 3. 并发控制

```typescript
if (inProgressCount > 1) {
  throw new Error("Only one task can be in_progress at a time");
}
```

**原因：**
- 单焦点原则：一次只做一件事
- 降低复杂性：避免并发冲突
- 提高成功率：专注完成当前任务

class TodoManager {
  constructor(private filePath: string = ".todos.json") {
    this.load();

  if (!item.depends_on) return true;

## 总结

### 核心学习点

1. **状态追踪**: Agent 可以通过工具维护自己的内部状态
2. **约束设计**: 通过验证逻辑保证状态一致性
3. **提醒模式**: 检测模型行为偏差并注入修正信号
4. **可视化**: 将内部状态渲染为人类可读格式

### 设计原则

- **单一入口**: 状态只能通过特定工具修改
- **显式状态**: 状态转换必须明确标记
- **容错机制**: 错误不中断，返回给 LLM 处理
- **适度提醒**: 不打扰正常流程，但防止遗忘
