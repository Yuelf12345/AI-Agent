# LangChain Tool Use 学习笔记

## 概述

本文通过分析 `2.tool_use.ts` 文件，学习如何在 Agent 循环中添加工具系统。**核心洞察：循环本身没有变化，只是添加了工具。**

## 核心架构

### Agent 循环 + 工具分发

```
+----------+      +-------+      +------------------+
|   User   | ---> |  LLM  | ---> | Tool Dispatch    |
|  prompt  |      |       |      | {                |
+----------+      +---+---+      |   bash: run_bash |
                      ^          |   read: run_read |
                      |          |   write: run_wr  |
                      +----------+   edit: run_edit |
                      tool_result| }                |
                                 +------------------+
```

### 与 s01 的区别

| 方面 | s01 (loop) | s02 (tool_use) |
|------|------------|----------------|
| Agent 循环 | ✅ 基础循环 | ✅ **完全相同** |
| 工具数量 | 2个 (bash, weather) | 4个 (bash, read, write, edit) |
| 工具分发 | if-else 分发 | **分发映射表** |
| 安全机制 | 无 | **路径安全验证** |
| 输出截断 | 无 | **50000 字符限制** |

## 代码结构分析

### 1. 工作目录与路径安全

```typescript
const WORKDIR = process.cwd();

// 安全路径验证：防止路径逃逸
function safePath(p: string): string {
  const resolved = path.resolve(WORKDIR, p);
  if (!resolved.startsWith(WORKDIR)) {
    throw new Error(`Path escapes workspace: ${p}`);
  }
  return resolved;
}
```

**安全检查：**
- 所有路径先 `resolve` 为绝对路径
- 检查是否以 `WORKDIR` 开头
- 阻止 `../` 等路径逃逸攻击

### 2. 工具实现层

#### Bash 工具

```typescript
function runBash(command: string): string {
  // 危险命令黑名单
  const dangerous = ["rm -rf /", "sudo", "shutdown", "reboot", "> /dev/"];
  if (dangerous.some((d) => command.includes(d))) {
    return "Error: Dangerous command blocked";
  }
  
  try {
    const result = execSync(command, {
      cwd: WORKDIR,
      encoding: "utf-8",
      timeout: 120000,  // 2分钟超时
    });
    return result.slice(0, 50000) || "(no output)";
  } catch (error: any) {
    return `Error: ${error.message}`;
  }
}
```

**安全机制：**
- 黑名单过滤危险命令
- 超时保护（120秒）
- 输出截断（50000字符）

#### Read 工具

```typescript
function runRead(filePath: string, limit?: number): string {
  try {
    const safe = safePath(filePath);  // 路径安全检查
    const content = fs.readFileSync(safe, "utf-8");
    const lines = content.split("\n");
    
    // 行数限制
    if (limit && limit < lines.length) {
      return lines
        .slice(0, limit)
        .concat([`... (${lines.length - limit} more lines)`])
        .join("\n")
        .slice(0, 50000);
    }
    return content.slice(0, 50000);
  } catch (error: any) {
    return `Error: ${error.message}`;
  }
}
```

#### Write 工具

```typescript
function runWrite(filePath: string, content: string): string {
  try {
    const safe = safePath(filePath);
    const dir = path.dirname(safe);
    
    // 自动创建目录
    if (!fs.existsSync(dir)) {
      fs.mkdirSync(dir, { recursive: true });
    }
    
    fs.writeFileSync(safe, content);
    return `Wrote ${content.length} bytes to ${filePath}`;
  } catch (error: any) {
    return `Error: ${error.message}`;
  }
}
```

#### Edit 工具

```typescript
function runEdit(filePath: string, oldText: string, newText: string): string {
  try {
    const safe = safePath(filePath);
    const content = fs.readFileSync(safe, "utf-8");
    
    // 精确匹配检查
    if (!content.includes(oldText)) {
      return `Error: Text not found in ${filePath}`;
    }
    
    // 只替换第一个匹配项
    const newContent = content.replace(oldText, newText);
    fs.writeFileSync(safe, newContent);
    return `Edited ${filePath}`;
  } catch (error: any) {
    return `Error: ${error.message}`;
  }
}
```

### 3. 工具定义层

使用 LangChain 的 `tool()` 函数包装：

```typescript
const bashTool = tool(
  (input: { command: string }) => runBash(input.command),
  {
    name: "bash",
    description: "Run a shell command.",
    schema: {
      type: "object",
      properties: { command: { type: "string" } },
      required: ["command"],
    },
  }
);

const readTool = tool(
  (input: { path: string; limit?: number }) => runRead(input.path, input.limit),
  {
    name: "read_file",
    description: "Read file contents.",
    schema: {
      type: "object",
      properties: {
        path: { type: "string" },
        limit: { type: "number" },
      },
      required: ["path"],
    },
  }
);
```

**工具定义三要素：**
| 要素 | 说明 |
|------|------|
| `name` | 工具唯一标识，LLM 调用时使用 |
| `description` | 工具描述，LLM 根据此描述决定是否调用 |
| `schema` | JSON Schema 定义参数结构 |

### 4. 工具绑定

```typescript
const llm = new ChatOpenAI({
  model: "qwen-plus",
  temperature: 0.5,
  maxTokens: 8000,
});

const llmWithTools = llm.bindTools([bashTool, readTool, writeTool, editTool]);
```

`bindTools()` 将工具定义附加到 LLM，使其在推理时知道可用的工具。

### 5. 工具分发映射表

```typescript
const TOOL_HANDLERS: Record<string, (args: Record<string, any>) => string> = {
  bash: (args) => runBash(args.command),
  read_file: (args) => runRead(args.path, args.limit),
  write_file: (args) => runWrite(args.path, args.content),
  edit_file: (args) => runEdit(args.path, args.old_text, args.new_text),
};
```

**相比 s01 的 if-else 分发：**
- 更易扩展：添加新工具只需增加映射条目
- 更清晰：工具名与处理函数一一对应
- 更解耦：分发逻辑与实现分离

### 6. Agent 循环（几乎不变）

```typescript
const agentLoop = async (messages: BaseMessage[]) => {
  while (true) {
    const response = await llmWithTools.invoke(messages);
    messages.push(new AIMessage({ content: response.content }));
    
    // 检查是否需要调用工具
    if (response.response_metadata?.finish_reason !== "tool_calls") {
      return;  // 没有工具调用，结束循环
    }

    // 执行工具调用（使用分发映射表）
    const results = response.tool_calls?.map((toolCall) => {
      const handler = TOOL_HANDLERS[toolCall.name];
      const output = handler 
        ? handler(toolCall.args) 
        : `Unknown tool: ${toolCall.name}`;

      console.log(`> ${toolCall.name}:`);
      console.log(output.slice(0, 200));

      return new ToolMessage({
        tool_call_id: toolCall.id!,
        content: output,
      });
    }) || [];

    messages.push(...results);
  }
};
```

**唯一变化：** 使用 `T OOL_HANDLERS[toolCall.name]` 替代 if-else 分发。

## 关键设计模式

### 1. 双层架构

```
┌─────────────────┐
│   Tool 定义层    │  ← LangChain tool() 包装
│   (tool.ts)      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   Tool 实现层    │  ← 纯函数，无依赖
│   (run_xxx)      │
└─────────────────┘
```

**好处：**
- 实现层可独立测试
- 定义层可切换框架
- 职责分离，易于维护

### 2. 安全防御策略

| 防御层 | 机制 | 目标 |
|--------|------|------|
| 路径安全 | `safePath()` | 防止路径逃逸 |
| 命令安全 | 黑名单过滤 | 阻止危险命令 |
| 资源安全 | 输出截断 | 防止内存溢出 |
| 时间安全 | 超时保护 | 防止无限等待 |

### 3. 错误处理策略

```typescript
try {
  // 执行操作
  return result;
} catch (error: any) {
  return `Error: ${error.message}`;  // 返回错误信息给 LLM
}
```

**设计理念：** 错误信息返回给 LLM，让模型决定如何处理，而不是直接抛出异常。

## 工具选择机制

模型根据以下信息选择工具：

```
用户问题 ──▶ 模型推理 ──▶ 匹配工具描述 ──▶ 选择最佳工具
                │
                ├── "读取文件内容" ──▶ read_file
                ├── "执行命令" ──▶ bash
                ├── "创建文件" ──▶ write_file
                └── "修改文件" ──▶ edit_file
```

**关键因素：**
1. 用户问题语义
2. 工具 `description` 描述
3. 工具参数 `schema` 约束

## 总结

### 核心学习点

1. **循环不变性**: 添加工具不需要修改 Agent 循环逻辑
2. **分发模式**: 使用映射表替代 if-else，更易扩展
3. **安全优先**: 路径验证、命令过滤、输出截断、超时保护
4. **错误返回**: 错误信息返回给 LLM，让模型决定下一步

### 设计原则

- **简单**: 循环保持简单，复杂度在工具层
- **安全**: 多层防御，永不信任输入
- **可扩展**: 添加工具只需增加映射，不改循环
- **可观测**: 日志记录所有工具调用
