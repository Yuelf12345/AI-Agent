# LangChain Agent Loop 学习笔记

## 概述

本文通过分析 `1.loop.ts` 文件，学习 LangChain 中 Agent 循环的核心实现模式。这是理解 AI Agent 工作原理的基础。

## 核心概念：Agent 循环

Agent 循环的本质是一个简单的模式：

```
while stop_reason == "tool_use":
    response = LLM(messages, tools)
    execute tools
    append results
```

用流程图表示：

```
+----------+      +-------+      +---------+
|   User   | ---> |  LLM  | ---> |  Tool   |
|  prompt  |      |       |      | execute |
+----------+      +---+---+      +----+----+
                      ^               |
                      |   tool_result |
                      +---------------+
                      (loop continues)
```

## 代码结构分析

### 1. 定义工具函数

```typescript
// Bash 命令执行工具
function runBash(command: string): string {
  try {
    return execSync(command, { encoding: "utf-8" });
  } catch (error) {
    return `Error: ${error}`;
  }
}

// 天气查询工具（模拟）
function getWeather(city: string): string {
  const weatherData: Record<string, string> = {
    "北京": "晴天，15°C",
    "上海": "多云，18°C",
    "广州": "小雨，22°C",
    "深圳": "晴天，25°C",
  };
  return weatherData[city] || `${city}：暂无天气数据`;
}
```

### 2. 创建工具定义

使用 LangChain 的 `tool()` 函数包装工具：

```typescript
const bashTool = tool(runBash, {
  name: "bash",
  description: "Run a shell command.",
  schema: {
    type: "object",
    properties: { command: { type: "string" } },
    required: ["command"],
  },
});

const weatherTool = tool(getWeather, {
  name: "get_weather",
  description: "查询指定城市的天气信息",
  schema: {
    type: "object",
    properties: { city: { type: "string", description: "城市名称" } },
    required: ["city"],
  },
});
```

**关键点：**
- `name`: 工具的唯一标识符
- `description`: 告诉模型工具的用途，模型根据描述决定何时调用
- `schema`: 定义工具参数的 JSON Schema

### 3. 绑定工具到 LLM

```typescript
const llm = new ChatOpenAI({
  model: "qwen-plus",
  temperature: 0.5,
  maxTokens: 1000,
});

const llmWithTools = llm.bindTools([bashTool, weatherTool]);
```

`bindTools()` 让模型知道有哪些工具可用，但不会自动执行。

### 4. Agent 循环核心

```typescript
const agentLoop = async (messages: BaseMessage[]) => {
  while (true) {
    // 1. 调用 LLM
    const response = await llmWithTools.invoke(messages);
    
    // 2. 记录 AI 回复
    messages.push(new AIMessage({ content: response.content }));

    // 3. 检查是否需要调用工具
    if (response.response_metadata?.finish_reason !== "tool_calls") {
      return; // 没有工具调用，结束循环
    }

    // 4. 执行工具调用
    const results = response.tool_calls?.map((toolCall) => {
      let output: string;
      if (toolCall.name === "bash") {
        output = runBash(toolCall.args.command as string);
      } else if (toolCall.name === "get_weather") {
        output = getWeather(toolCall.args.city as string);
      } else {
        output = `未知工具: ${toolCall.name}`;
      }
      
      return new ToolMessage({
        tool_call_id: toolCall.id!,
        content: output,
      });
    }) || [];

    // 5. 将工具结果加入消息历史
    messages.push(...results);
  }
};
```

**循环流程：**
1. 调用 LLM，获取响应
2. 将 AI 响应加入消息历史
3. 检查 `finish_reason`，如果不是 `"tool_calls"`，结束循环
4. 执行所有工具调用，获取结果
5. 将工具结果加入消息历史
6. 回到步骤 1，继续循环

### 5. REPL 交互界面

```typescript
const main = async () => {
  const history: BaseMessage[] = [];

  while (true) {
    const query = await question("\x1b[36ms01 >> \x1b[0m");
    
    if (cmd === "q" || cmd === "exit" || cmd === "") {

    history.push(new HumanMessage({ content: query }));
    await agentLoop(history);
    
    // 打印最终结果
    const responseContent = history[history.length - 1]?.content;
    if (typeof responseContent === "string") {
      console.log(responseContent);
    }
  }

  rl.close();
};
```

## 工具选择机制

模型根据以下因素选择工具：
1. **用户问题内容** - 问天气会选择 `get_weather`，问系统操作会选择 `bash`
2. **工具描述** - 模型根据 `description` 判断哪个工具更适合

示例：
- "北京天气怎么样？" → 选择 `get_weather`
- "列出当前目录文件" → 选择 `bash`

## 关键学习点

### 1. 消息类型
- `HumanMessage`: 用户输入
- `AIMessage`: 模型响应
- `ToolMessage`: 工具执行结果

### 2. finish_reason 值
- `"tool_calls"`: 模型请求调用工具
- `"stop"`: 模型正常结束
- 其他值根据不同模型可能有所不同

### 3. bindTools vs createAgent
- `bindTools`: 低级别 API，手动控制循环，适合学习原理
- `createAgent`: 高级别 API，自动处理循环，适合生产使用

## 扩展方向

1. **添加更多工具**: 如搜索、数据库查询等
2. **错误处理**: 工具执行失败时的重试机制
3. **并行执行**: 同时执行多个工具调用
4. **状态管理**: 会话持久化、上下文窗口管理

## 总结

Agent 循环是 AI Agent 的核心模式：**不断调用 LLM，执行工具，将结果反馈给 LLM，直到任务完成**。理解这个循环是构建复杂 Agent 系统的基础。
