# 学完 LangChain/LangGraph 后的进阶路线

> 你已掌握：Prompt 工程、LangChain、LangGraph
> 目标：从"会用"到"精通"，构建生产级 AI 应用

---

## 方向一：RAG 高级技术

LangChain 的 RAG 只是入门，生产级系统需要这些：

- [ ] **分块策略**：语义分块、递归分块、Agentic 分块
- [ ] **重排序（Reranking）**：ColBERT、Cross-encoder
- [ ] **查询改写**：HyDE、查询扩展、多查询检索
- [ ] **混合检索**：向量 + 关键词 + 图谱
- [ ] **图谱 RAG**：结合知识图谱（Neo4j + LangChain GraphVectorStore）

**实践项目**：做一个能处理 1000+ 文档的知识库，目标召回率 90%+

---

## 方向二：多 Agent 系统（发挥 LangGraph 优势）

LangGraph 就是为复杂 Agent 工作流设计的：

- [ ] **StateGraph 高级模式**：条件边、循环、并行分支
- [ ] **Supervisor 模式**：一个协调者 Agent + 多个专家 Agent
- [ ] **网状协作**：Agent 间直接通信（无需中心协调）
- [ ] **层级规划**：Planning Agent → Execution Agents
- [ ] **持久化**：Checkpointer、Memory 管理、状态恢复

**实践项目**：做一个「产品经理 → 架构师 → 程序员」的 Multi-Agent 协作链

---

## 方向三：生产化工程化（最实用）

Agent 跑通容易，量产很难：

- [ ] **评估体系**：RAGAS、LangSmith 评测指标设计
- [ ] **可观测性**：Token 消耗、延迟、错误率追踪
- [ ] **成本控制**：模型路由（简单任务 4o-mini，复杂任务 GPT-4）
- [ ] **安全机制**：Prompt 注入防护、输出审核、敏感信息过滤
- [ ] **A/B 测试**：不同 Prompt/模型的效果对比

**实践项目**：给现有 Agent 加上完整的观测、评估和成本监控体系

---

## 方向四：前端 AI 应用（发挥你的优势）

结合前端背景的差异化竞争力：

- [ ] **实时多模态**：Whisper + TTS + 实时视频流
- [ ] **AI 原生界面**：Vercel AI SDK 流式 UI、Generative UI
- [ ] **浏览器 AI**：WebGPU 推理、Transformers.js 本地运行
- [ ] **AI 辅助编程**：Cursor 插件开发、Code Interpreter 集成
- [ ] **可视化编排**：LangGraph 的图形化工作流编辑器

**实践项目**：做一个有实时语音交互 + 可视化工作流的 AI 助手

---

## 方向五：模型微调与本地化（进阶）

从调用 API 到拥有自己的模型：

- [ ] **高效微调**：LoRA、QLoRA（推荐 unsloth 库）
- [ ] **本地部署**：Ollama、vLLM、Llama.cpp
- [ ] **模型量化**：GGUF 格式，70B 模型在 24G 显存运行
- [ ] **知识蒸馏**：小模型学大模型能力
- [ ] **Transformers.js**：浏览器端运行小模型

**注意**：资源消耗大，初期用 API 足够，有需要时再深入
