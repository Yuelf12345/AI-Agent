# 四大上下文问题与解决策略

## 四大上下文问题

### 1. Context Poisoning（上下文污染）

**定义**：错误、有害或低质量信息进入上下文，"毒害"模型的理解和输出。

**表现**：
- 用户输入的错误信息被模型采纳
- 早期对话中的错误推理被后续内容延续
- 幻觉内容污染整个上下文环境

### 2. Context Distraction（上下文干扰）

**定义**：大量无关或低价值信息分散模型的注意力，导致核心任务被稀释。

**表现**：
- 模型被冗余信息分散注意力
- 关键指令被大量噪音淹没
- 输出偏离核心目标，纠结于次要细节

### 3. Context Confusion（上下文混淆）

**定义**：相似但不同的概念、实体或指令在上下文中相互干扰，导致模型无法正确区分。

**表现**：
- 不同实体被错误地关联或混淆
- 多个任务或角色的指令相互干扰
- 相似术语的歧义引发理解错误

### 4. Context Clash（上下文冲突）

**定义**：上下文中存在相互矛盾的信息、指令或约束，导致模型陷入两难。

**表现**：
- 先后指令相互矛盾
- 用户请求与系统约束冲突
- 多个有效规则之间存在逻辑冲突

---

## 四大解决策略

### 1. Write（写入）

**核心思想**：主动、显式地将关键信息写入上下文，而非依赖模型隐式推断。

**方法**：
- 使用显式声明定义重要规则和约束
- 采用结构化格式（如 Markdown 表格、编号列表）增强可读性
- 设置"上下文锚点"标记关键信息位置

**解决的问题**：
| 问题 | 解决方式 |
|------|---------|
| ✅ Context Poisoning | 明确写入正确信息，覆盖可能存在的错误内容 |
| ✅ Context Confusion | 显式定义和区分易混淆的概念与实体 |

---

### 2. Select（选择）

**核心思想**：精心筛选进入上下文的信息，只保留与当前任务高度相关的内容。

**方法**：
- 建立信息相关性评分机制
- 过滤低价值或无关的历史对话
- 按任务优先级动态调整上下文内容

**解决的问题**：
| 问题 | 解决方式 |
|------|---------|
| ✅ Context Distraction | 移除干扰信息，保持上下文聚焦 |
| ✅ Context Poisoning | 排除已识别的错误或有害信息 |

---

### 3. Compress（压缩）

**核心思想**：对上下文信息进行语义压缩，在保留核心含义的前提下减少 token 消耗。

**方法**：
- 使用摘要技术压缩长文本
- 采用缩写、符号等紧凑表达方式
- 删除冗余和重复信息
- 将低频细节转换为高层抽象

**解决的问题**：
| 问题 | 解决方式 |
|------|---------|
| ✅ Context Distraction | 压缩噪音信息，降低干扰权重 |
| ✅ Context Confusion | 去除造成混淆的冗余细节 |

---

### 4. Isolate（隔离）

**核心思想**：将相互影响的信息源或任务流程物理/逻辑隔离，避免交叉污染。

**方法**：
- 使用独立的 system prompt 区分不同任务
- 采用多代理架构分离职责
- 利用工具调用隔离不确定的推理路径
- 设置明确的上下文边界标记

**解决的问题**：
| 问题 | 解决方式 |
|------|---------|
| ✅ Context Clash | 物理隔离冲突源，避免同上下文并存 |
| ✅ Context Confusion | 逻辑隔离易混淆的概念域 |
| ✅ Context Poisoning | 隔离可疑或未经确认的信息源 |

|------------|:---------:|:-----------:|:---------:|:-----:|

## 实践建议

### 问题识别优先
在应用策略前，先诊断当前上下文存在的主要问题类型：
- 输出错误 → 检查 Poisoning
- 偏离目标 → 检查 Distraction
- 实体混淆 → 检查 Confusion
- 逻辑矛盾 → 检查 Clash

### 组合策略应用
单一问题可使用对应策略，复杂场景需组合应用：

| 场景 | 推荐策略组合 |
|------|-------------|
| 多轮对话质量下降 | Select + Compress |
| 多任务切换混乱 | Isolate + Write |
| 长文档理解偏差 | Compress + Select |
| 知识库检索不准确 | Write + Isolate |

### 策略执行时机
- **Write**：在对话开始时、关键转折点时主动执行
- **Select**：加载上下文前、评估信息源时执行
- **Compress**：上下文接近 token 限制时、总结阶段时执行
- **Isolate**：设计系统架构时、引入新信息源时执行

## 总结

| 策略 | 核心动作 | 一句话描述 |
|------|---------|-----------|
| **Write** | 主动写入 | "重要的东西，写下来" |
| **Select** | 精心筛选 | "只留有用的" |
| **Compress** | 语义压缩 | "保留精华，去掉冗余" |
| **Isolate** | 物理隔离 | "不能共存的，分开处理" |

四大策略相互配合，构成了上下文工程的核心工具箱。理解每个策略的适用场景和组合方式，是构建高质量 AI 应用的重要能力。
